# Database Schema — Facial Recognition & Access Management (Felicia)

Tables owned by the access-management feature. Defined with Sequelize (PostgreSQL,
`pgvector` extension for face embeddings).

## `users`
Core identity, role, registration-code logic, and biometric vector.

| Field | Type | Constraints |
|-------|------|-------------|
| id | INTEGER | PK, auto-increment |
| name | VARCHAR(100) | NOT NULL |
| email | VARCHAR(100) | NOT NULL, UNIQUE |
| password | VARCHAR(100) | NOT NULL (bcrypt hash) |
| role | ENUM('FM','Tenant','Staff') | default 'Tenant' |
| companyCode | VARCHAR(50) | UNIQUE, nullable |
| codeCreatedAt | TIMESTAMP | nullable (48h expiry clock) |
| codeMaxUsage | INTEGER | NOT NULL, default 10 |
| codeCurrentUsage | INTEGER | NOT NULL, default 0 |
| managerId | INTEGER | FK → users.id (self-reference) |
| isEnrolled | BOOLEAN | NOT NULL, default false |
| faceVector | VECTOR(512) | nullable (pgvector) |
| isActive | BOOLEAN | NOT NULL, default true |
| createdAt / updatedAt | TIMESTAMP | auto |

Relationships: self-reference `managerId` (Manager → StaffMembers); `users` 1—M `attendance`.

## `security_logs`
Access events and intrusion alerts shown on the V-Patrol dashboard.

| Field | Type | Constraints |
|-------|------|-------------|
| id | VARCHAR | PK |
| time | VARCHAR | NOT NULL |
| type | VARCHAR | NOT NULL (e.g. Access Granted, Intrusion) |
| desc | TEXT | NOT NULL |
| severity | VARCHAR | NOT NULL, default 'safe' |
| icon | VARCHAR | NOT NULL |
| personnelName | VARCHAR | nullable (soft link → users.name) |
| createdAt / updatedAt | TIMESTAMP | auto |

## `attendance`
IN/OUT events generated by gate recognition.

| Field | Type | Constraints |
|-------|------|-------------|
| id | INTEGER | PK |
| type | ENUM('IN','OUT') | NOT NULL |
| timestamp | TIMESTAMP | default NOW |
| userId | INTEGER | FK → users.id, ON DELETE CASCADE |
| createdAt / updatedAt | TIMESTAMP | auto |

## `invites`
Tenant onboarding invitation codes.

| Field | Type | Constraints |
|-------|------|-------------|
| id | INTEGER | PK |
| code | VARCHAR | NOT NULL, UNIQUE |
| role | ENUM('Tenant') | default 'Tenant' |
| isUsed | BOOLEAN | default false |
| expiresAt | TIMESTAMP | NOT NULL |

## `staff_members`
Legacy/auxiliary face store (soft-deletable).

| Field | Type | Constraints |
|-------|------|-------------|
| id | INTEGER | PK |
| name | VARCHAR(100) | NOT NULL |
| role | VARCHAR(50) | nullable |
| face_embedding | TEXT | NOT NULL |
| deletedAt | TIMESTAMP | soft delete (paranoid) |

> `bookings` and `incident_logs` exist in the shared DB but are owned by teammates
> (logistics / object-detection); see the full `er-diagram.md`.
